
🚀 Prompt Maestro: STEM Home Lab (IBM Bob Edition)

Contexto del Socio: Actúa como IBM Bob, mi socio de desarrollo de IA especializado en sistemas reales y gestión de repositorios complejos. Nuestra misión es crear una plataforma educativa de experimentos STEM para niños, donde la seguridad técnica sea una restricción lógica inquebrantable del software.

1. Definición de la Arquitectura del Repositorio
Diseña la estructura de carpetas y archivos necesaria para un proyecto escalable (Frontend en React/Next.js y Backend en Node.js/Python), integrando:


Directorio /experiments: Donde cada experimento es un archivo .md o .json con metadatos estrictos.

Esquema de Metadatos: Cada entrada debe incluir:


Nivel: Básico, Intermedio, Avanzado.


Semáforo de Seguridad: 🟢 (Autónomo), 🟡 (Supervisión sugerida), 🔴 (Supervisión obligatoria).


Hazard Tags: Categorías como chemical_reaction, electrical_circuit, high_temperature, sharp_objects.


EPP Requerido: Lista obligatoria de equipo de protección (gafas, guantes, etc.).

2. Lógica de Seguridad y "Guardrails" (El Hack)
Utiliza tu capacidad de razonamiento lógico para implementar las siguientes funciones en el código:


Adult Gate (Desafío Lógico): Los experimentos marcados como 🔴 deben permanecer bloqueados detrás de un componente que requiera resolver una ecuación o un reto de lógica para adultos.


Simulación de Seguridad Obligatoria: Antes de mostrar los pasos, genera un flujo donde el usuario deba identificar los riesgos potenciales en una imagen o descripción.


Modo "Científico Solo": Un filtro de estado que oculte dinámicamente todo el contenido que no sea 🟢.

3. Generación de Contenido y Adquisición
Para cada experimento que generes, sigue este estándar de documentación:


Lista de Materiales y Guía de Compra: Clasifica los materiales según su origen: Casa (reciclados), Supermercado, Farmacia o Ferretería.


Instrucciones con Seguridad Embebida: Los pasos críticos deben incluir advertencias visuales generadas por el sistema basadas en los Hazard Tags.


Explicación STEM: Un resumen final que explique la ciencia detrás del experimento de forma sencilla.

4. Tarea Inicial
Como primer paso, ayúdame a estructurar el archivo README del repositorio y el esquema JSON que usaremos para validar los experimentos. Luego, genera el primer experimento de prueba: un Electroimán Casero (Nivel: Intermedio, Supervisión: 🟡).
